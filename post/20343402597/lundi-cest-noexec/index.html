Lundi c'est Noexec

gem install noexec

Arrêtons d’utiliser bundle exec

Aujourd’hui une gem bien pratique dont toute l’utilité est dans le titre. Elle permet de lancer vos commandes et d’exécuter vos scripts dans le contexte de votre Gemfile, s’il y en a un dans le répertoire en cours, sans ajouter la commande :

bundle exec

Mais pour que cela fonctionne il faut ajouter la ligne suivante à votre .profile : 

RUBYOPT="-r`noexec`"

Une fois que c’est fait, à chaque fois que Ruby sera utilisé, il vérifiera si un Gemfile est présent pour fonctionner dans son contexte.

Voilà c’est tout pour aujourd’hui !
